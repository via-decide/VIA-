VIA Creator conviction
Creator flow

VIA creators should publish like a modular web, not like a trapped SPA tab.

The creator architecture proves the pattern: separate pages, shared runtime, direct page links, and browser-backed continuity. The rest of VIA now follows the same rule.

Belief

Direct surfaces are more reliable

Every major step in the creator journey should be URL-addressable, reload-safe, and easy to share. Your story page is not dependent on one live router process staying healthy.

Result

Shared state still survives

Profiles, drafts, and local posts stay available because the runtime writes to localStorage and shared helpers instead of ephemeral tab state.

Flow

What happens next

  1. Open creator onboarding to define your publishing identity.
  2. Save your profile once and let every VIA page reuse it.
  3. Return directly to feed, profile, or agent pages with no router handoff.
